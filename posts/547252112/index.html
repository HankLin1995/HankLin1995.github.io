工程缺失追蹤系統

發表於 2025-10-08 分類於 專案開發 閱讀次數：

前言

最近完成了一套缺失追蹤管理系統，主要是為了解決工程驗收階段的缺失管理問題，想跟大家分享一下開發過程和使用心得。

痛點

做工程的朋友應該都遇過這些問題：

  1. Excel 表格管理缺失，多人編輯版本會亂掉，不知道哪個是最新的
  2. LINE 群組溝通位置，「3F 走廊靠近電梯」講半天廠商還是找不到
  3. 照片散落各處，手機相簿、LINE 訊息、電腦資料夾，要找對比照片很麻煩
  4. 進度追蹤困難，每天要打一堆電話問廠商處理到哪裡
  5. 驗收沒有依據，修繕前後照片對不起來，驗收時容易有爭議

如果沒有好好管理這些缺失，到了業主驗收或督導檢查的時候會很可怕的...

解決方案

開發一套專門針對工程缺失管理的系統，主要解決幾個核心問題：

  • 廠商協作要簡單 - 不用註冊帳號，傳連結就能用
  • 位置標示要清楚 - 直接在平面圖上點選標記
  • 流程追蹤要完整 - 從建立到驗收每個步驟都有記錄
  • 數據分析要實用 - 廠商績效、缺失類型統計一目了然

系統核心功能

1. 唯一碼機制

每個缺失建立後系統會自動產生一組唯一識別碼，廠商不用註冊帳號，直接點連結就能看到缺失詳情和回報修繕狀況。

操作流程：

  1. 在系統建立缺失
  2. 複製系統產生的連結
  3. 傳給廠商（LINE、Email 都可以）
  4. 廠商點開連結就能看到缺失詳情並回報

這樣廠商不用記一堆帳號密碼，我們也不用花時間教他們怎麼操作系統。

2. 圖面標記功能

上傳平面圖後可以直接在圖面上點選標記缺失位置，廠商打開連結就能看到精確的位置標示。

好處：

  • 不用再口頭描述位置
  • 廠商不會找錯地方
  • 一張圖面可以看到所有缺失分布

以前要跟廠商解釋「3F 走廊靠近電梯」在哪裡，現在直接看圖面標記就知道了。

3. 狀態追蹤機制

系統會自動追蹤每個缺失的處理狀態：

廠商提交修繕報告後，系統會自動通知管理者進行驗收，不用再逐一打電話詢問進度。

操作流程

實際使用起來很簡單：

基本資料建立

  1. 建立工程名稱
  2. 建立平面圖
  3. 建立夥伴
  4. 建立廠商名稱
  5. 建立缺失分類

管理者建立缺失

  1. 填寫缺失資訊（描述、分類、指派廠商）
  2. 在平面圖上點擊標記位置
  3. 上傳現場照片
  4. 複製連結傳給廠商

廠商回報修繕

  1. 點擊連結查看缺失詳情
  2. 前往現場處理
  3. 完成後填寫修繕說明、上傳照片
  4. 點擊「確認修繕」

管理者驗收

  1. 收到「待確認」通知
  2. 查看修繕前後照片對比
  3. 決定「確認結果」或「退回重辦」

整個流程下來，系統操作時間大概 6 分鐘就搞定了。

其他功能

篩選搜尋

  • 可以依狀態、分類、廠商篩選缺失
  • 關鍵字搜尋功能
  • 緊急程度自動標示（逾期會變紅色）

數據統計

  • 缺失處理進度統計
  • 廠商績效分析
  • 缺失類型分布圖表

使用心得

導入建議

  1. 一次建檔完整 - 建議一開始就把所有缺失建檔完成，雖然要花一些時間，但建完之後就輕鬆了
  2. 圖面要清晰 - 上傳高解析度的平面圖，標記才會精確
  3. 廠商要溝通 - 第一次使用時，簡單跟廠商說明如何操作
  4. 持續使用 - 不要中途放棄回到舊方法，堅持使用才能看到效益

快速參考

常用操作

功能 操作路徑
建立專案 工程列表 → 新增工程
上傳圖面 工程詳情 → 圖面管理 → 新增圖面
建立缺失 缺失 → 缺失表單 → 填寫資訊 → 建立
通知廠商 缺失列表 → 選擇缺失 → 複製連結 → 傳給廠商
驗收修繕 缺失列表 → 篩選「待確認」→ 點擊確認
查看統計 統計 → 儀表板
